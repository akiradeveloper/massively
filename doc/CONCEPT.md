## Architecture

Build a library like NVIDIA Thrust (https://github.com/NVIDIA/thrust) on top of
CubeCL (https://github.com/tracel-ai/cubecl).

## Data

### DeviceVec

Represents a data region on the GPU.
Transfers are explicit.

- `to_device`: CPU-to-GPU transfer
- `to_host`: GPU-to-CPU transfer

### DeviceSlice / DeviceSliceMut

- Immutable slice: `DeviceVec::slice<R: RangeBounds>(&self, range: R) -> DeviceSlice`
- Mutable slice: `DeviceVec::slice_mut<R: RangeBounds>(&self, range: R) -> DeviceSliceMut`
- Slices can create further slices:
  - `DeviceSlice::slice -> DeviceSlice`
  - `DeviceSliceMut::slice -> DeviceSlice`
  - `DeviceSliceMut::slice_mut -> DeviceSliceMut`

### SoA (Structure of Arrays)

For GPU computation, SoA is generally more performant than AoS.
Group `DeviceSlice`s into tuples, turn them into an `MIter`, and use them for
computation.

- `DeviceSlice<T>: MIter<Item = T>`
- `Zip(MIter<A>, MIter<B>): MIter<Item = (A,B,...)>`
- `Zip(DeviceSliceMut<A>, DeviceSliceMut<B>): MIterMut<Item = (A,B)>`

## Algorithms

All functions are described in abstract form.

### Notation

- `&[a]` means `MIter<Item = a>`.
- `&mut[a]` means `MIterMut<Item =a>`.
- `T?` means `Option<T>`.
- `idx` means `MIndex`.
